# PRD — 고객 문의 기록 관리 웹앱

> 버전 0.1 · 2026-06-27

---

## 한 줄 목적

B2B SaaS 운영자가 고객 문의를 접수부터 완료까지 혼자 빠르게 추적·관리할 수 있는 브라우저 기반 경량 도구

---

## 사용자

| 항목 | 내용 |
|------|------|
| 인원 | 1명 (비개발자, 단독 운영) |
| 환경 | PC + 모바일 (반응형 필수) |
| 기술 수준 | 파일을 열어서 바로 쓸 수 있어야 함 (설치·서버 없음) |

---

## 핵심 기능 4개

### 1. 문의 CRUD
- 새 문의 추가, 목록 조회, 수정, 삭제
- 각 문의에 포함되는 필드:
  - 접수일 (날짜 선택)
  - 고객사명 + 담당자명
  - 연락처 (전화 또는 이메일)
  - 문의 유형 (기술지원 / 계약·결제 / 기능 요청 / 기타)
  - 문의 내용 (자유 입력)
  - 처리 메모 (내부용)

### 2. 상태 관리 — 4단계 워크플로
```
접수 → 확인 → 처리중 → 완료
```
- 각 단계는 색상 배지로 구분
- 목록에서 상태만 빠르게 바꿀 수 있어야 함

### 3. 검색 · 필터
- 고객사명 / 담당자명 / 문의 내용 통합 텍스트 검색
- 상태별 필터 (전체 / 접수 / 확인 / 처리중 / 완료)
- 접수일 최신순 정렬 (기본값)

### 4. CSV 내보내기
- 현재 필터 결과 또는 전체 기록을 CSV로 다운로드
- 엑셀에서 바로 열리도록 UTF-8 BOM 포함

## 화면 구성 (대략)

```
┌─────────────────────────────────────────────┐
│  문의관리              [+ 새 문의 추가]       │  ← 헤더 (fixed)
├─────────────────────────────────────────────┤
│  전체 12  │  접수 3  │  확인 2  │  처리중 4  │  완료 3  │  ← 통계 카드
├─────────────────────────────────────────────┤
│  🔍 검색…   [상태 ▾]  [유형 ▾]  [CSV ⬇]    │  ← 툴바
├────────┬─────────┬────────┬────────┬────────┤
│ 접수일 │ 고객사  │ 유형   │ 내용   │ 상태   │  ← 테이블 헤더
├────────┼─────────┼────────┼────────┼────────┤
│ ...    │ ...     │ ...    │ ...    │ [배지] │  ← 행 클릭 → 상세 모달
└────────┴─────────┴────────┴────────┴────────┘

[상세/수정 모달] — 오버레이로 표시, 외부 클릭 시 닫힘
```

**모바일**: 테이블 대신 카드형 목록으로 전환 (flex 스택)

## 저장 방식

| 항목 | 내용 |
|------|------|
| 방식 | 브라우저 `localStorage` |
| 포맷 | JSON 배열 (키: `cust_inquiries`) |
| 용량 | ~5MB 한도 (문의 수천 건은 여유롭게 저장 가능) |
| 주의 | 같은 브라우저·같은 파일 경로에서만 데이터 유지 |
| 백업 | CSV 내보내기로 수동 백업 가능 |

- 다국어 지원
